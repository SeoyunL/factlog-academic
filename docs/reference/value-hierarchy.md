# 값 계층 (`policy/value-hierarchy.md`)

> 🌐 [English](value-hierarchy.en.md) | **한국어**

같은 관계의 두 값은, 따로 말해 주지 않으면 서로 무관한 문자열입니다. 코호트연구는
관찰연구**입니다**. 그러나 선언이 없으면 `relation(P, "연구유형", "관찰연구")?` 는
정확히 `관찰연구` 라고 적힌 행만 돌려주고 `코호트연구` 로 기록된 행을 전부 놓칩니다 —
조용한 누락이고, 이 KB가 막으려는 바로 그 실패입니다. 실제 KB에서 그 질의는 14편 중
6편만 반환했습니다.

## 파일 문법

```markdown
# policy/value-hierarchy.md
- 연구유형: 코호트연구 ⊂ 관찰연구
- 연구유형: 단면연구 ⊂ 관찰연구
- 대상질환: `emphysema` <: COPD
```

한 줄에 선언 하나이며 형식은 `관계: 좁은값 ⊂ 넓은값` 입니다.

- `<:` 와 `<` 는 `⊂` 의 ASCII 표기입니다.
- `#` 주석과 `-`·`*` 불릿을 쓸 수 있습니다.
- 공백·`:`·`<` 가 든 값은 백틱으로 감싸십시오. 백틱으로 감싼 이름은 연산자를 찾기
  **전에** 떼어 내므로, 값 안의 `:` 나 `<` 가 구분자로 잘못 잘리지 않습니다.
- 관계명과 값은 NFC로 정규화해 읽습니다. macOS에서 NFD로 저장한 정책 파일이 NFC인
  accepted 사실과 어긋나 조용히 무동작이 되는 일을 막기 위한 것입니다.

조상은 이행적입니다 — `a ⊂ b` 이고 `b ⊂ c` 면 `c` 질의가 `a` 행도 잡습니다.

넓은 값이 어느 사실에도 등장하지 않아도 됩니다. 선언만으로 질의 가능한 개념이 됩니다.

`factlog init` 이 주석 처리된 예시와 함께 이 파일을 만들어 둡니다.

## 적용 범위 계약

포섭은 질의의 **객체(object)** 를 매칭할 때 적용되며, 게이트·평가기·논리 리포트 셋
모두가 같은 선언을 읽습니다. 그래서 `/factlog ask` 와 `/factlog check` 가 같은 질문에
다르게 답할 수 없습니다.

- **사실을 다시 쓰지 않습니다.** `accepted.dl` 은 여전히 승인된 후보 행의 1:1
  투영이고, 각 행은 자기 값과 자기 출처를 그대로 유지합니다.
- **단방향입니다.** 넓은 값을 물으면 좁은 값의 행이 돌아오지만, 좁은 값을 물으면 넓은
  값의 행은 돌아오지 않습니다.
- **적용되지 않는 곳**: `factlog search` · `provenance` · `vocab` · 커버리지 · 모순
  탐지. 이들은 여전히 값을 정확히 일치시킵니다.

## 순환과 경고

실수는 조용히 무동작으로 남지 않고 보고됩니다.

- **순환**은 통째로 폐기됩니다. 그대로 두면 포섭이 양방향이 되어 단방향 계약이
  깨집니다. 폐기된 값은 경고로 보고됩니다.
- 선언 형식이 아닌 줄(주석도 아니고 파싱도 안 되는 줄), 자기 자신을 부모로 선언한 줄은
  무시하되 줄 번호와 함께 경고합니다.
- 어떤 accepted 사실도 쓰지 않는 **관계**나 **값**을 선언하면 논리 리포트의 경고로
  뜹니다. 그러지 않으면 오타 하나로, 넓은 질의가 좁은 행을 잡고 있다고 **믿게**
  됩니다.

## 모순과의 관계

한 subject 에 `관찰연구` 와 `코호트연구` 가 둘 다 단언돼 있고 그 관계가
[단일값](single-valued.md)이면, 기본적으로는 `CONFLICT` 입니다. 그러나 두 값이
상위-하위유형이라면 어느 쪽도 틀리지 않았습니다 — 여기 관계를 선언하면 두 행이 모두
유지됩니다.
